Merchandise Management System

Proposed Software Process Model

The Cabezón Merchandise Management System is a software system designed to manage and track the inventories of the various stores and restaurants throughout the park. Since the park already has many systems in place as well as staff operating the park, it is imperative that the various needed systems are developed as fast as possible to get the park fully functional. To accomplish this task, an agile software development approach is needed.

Advantages of an Agile software development approach:

  • Stakeholders (park employees) have opportunities to test the MMS early and often, allowing for constant feedback so changes can be made rapidly.
  • Users are able to get accustomed to the software early on.
  • A basic, yet fully operational version of the MMS software can be developed very quickly so the park can be operational at a sooner date.
  • Development is able to be user-focused, giving the stakeholders a say in what changes are made.
  • Changes in user requirements are easily incorporated.
  • Predictable and scheduled releases.
  • Breaking the software into manageable units allows the developers to focus on testing, providing frequent builds, and improving the overall quality of the software.

Disadvantages of a Waterfall software development approach:

  • There is a lot of time taken to gather requirements and document the software during development.
  • Less predictable release timetable.
  • If there are any changes that must be made after the software is released, it is much harder to go back and make those changes.
  • There is very little (if any) user feedback during development, and incorporating user feedback later on becomes complicated.
  • Late testing of the software leads to more delays.
  • Very rigid and inflexible development schedule.

Use Cases

Add Store Location

To add a new store location, a staff member clicks on the “Add new location” button on the main page of the Merchandise Management System software. From there, he can enter in store details (title, description, etc.) as well as upload a predetermined store inventory from a spreadsheet or enter items individually.

Upload Store Inventory

If a store’s inventory ever needs to be changed on a large scale, a staff member can upload to the MMS a premade spreadsheet containing the inventory information. To do this, the employee clicks on a location to view that location’s details. From there, the employee can directly upload the new spreadsheet, which will update the database with the correct inventory information.

Update Item Details

When an item’s details need to be changed, a staff member can do that manually from the MMS. First, the employee selects the item to be changed. After clicking “Update,” he is then able to change the item’s price, name, location, or description.
