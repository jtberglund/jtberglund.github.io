Automated Tour System

Scenarios

Initial Assumption

The tour has already started, and the group of guests is making its way around the park in the tour vehicle.

Normal

The vehicle stops at every attraction, using the Heads-Up Surveillance system to point out surrounding flora and fauna. The tour guide for that vehicle is then able to give the guests additional information about that exhibits. If the guests have any questions, they are able to use the Interactive Communication system to contact a member of the park staff (such as an expert or handler of the exhibit the vehicle is currently stopped at) to ask any questions. The vehicle then resumes the tour, continuing this pattern through all exhibits until the tour has reached its conclusion.

What Can Go Wrong

The vehicle could break down. In this case, the passengers would be safely evacuated and a shuttle would come to pick them up and return them to the start of the tour, where they would be offered a free tour as compensation.

The Interactive Communication system could break. If this happens, the tour can still function as normal, with the on-board tour guide providing as much information as possible.

The Heads-Up Surveillance system could break. If this happened, the tour guide would have to take more time to point our surrounding flora and fauna so the passengers do not miss any exciting exhibits.

System State on Completion

The vehicle has returned to the start of the tour and all passengers have been safely evacuated.
